# V0.4 QAT Summary

日期：2026-05-21

QAT V0.4 的目标是把 HK416 主枪口 `BarrelPos` 迁移到 quick attachment transform，移除 XML 中按状态循环改枪口位置的逻辑。`BarrelPos` 只在 GunSmith 配置刷新、quick-slot 内容变化等低频路径更新；开火路径只读取已经写好的 `BarrelPos`。

## 已完成内容

- 新增 QAT barrel transform 注册路径。
  - Lua 侧通过 `DeepGunsmithClearQuickAttachmentBarrelTransforms` 清理当前物品的 barrel transform。
  - Lua 侧通过 `DeepGunsmithRegisterQuickAttachmentBarrelCanvasPoint` 注册 GunSmith canvas 上的枪口点和部件出口偏移。
  - C# 侧使用 composed world sprite state 转换到 item local 后立即写入当前 `RangedWeapon.BarrelPos`，不 patch `Use`，不在每次开火前写属性。
- 精简 barrel 配置模型。
  - 不再保留 `projectileIndex`。
  - 不再保留 `quickAttachmentBarrels`。
  - muzzle quick-slot path 由现有 quick slot binding 推算。
- 统一 Lua metadata 命名。
  - 删除 `muzzleLength`。
  - 改用 `quickAttachmentTransform.muzzleOutletOffset = { x = ..., y = ... }`。
  - 裸枪口位置由 `muzzle_mount` quick-slot anchor 推算，结构枪管本身不再配置 `muzzleOutletOffset`。
  - `Muzzle.lua` 提供枪口件相对挂点的附加出口偏移，短枪口为 `{ x = 30, y = 0 }`，长枪口/消音器为 `{ x = 90, y = 0 }`。
- 迁移 HK416 主枪口 XML。
  - 删除旧的 `StatusEffect barrelpos=234/264/324,26` 动态切换。
  - 主枪口 flash/spark 拆到 `statuseffecttags="Gunsmith_BarrelParticle"` 的 `StatusEffect`。
  - 主枪口 flash/spark 不再使用 `Distance`。
  - V0.4.1 后，下挂发射点也进入 QAT barrel path。
- 接入 HK416 下挂 barrel path。
  - QAT barrel registry 从单一 rule 扩展为 keyed rules：`primary` 和 `lower_rail`。
  - VCE `currentProjectileSelected` 通过可选动态 bridge 映射到当前 barrel rule：`0 -> primary`，`1 -> lower_rail`。
  - VCE 不存在时允许延迟到实际 item component 出现后再补 patch；VCE selector getter/setter 缺失时显式报错。
  - 若 selector 切到 `lower_rail` 但 lower rule 未注册，说明当前没有可发射下挂，QAT 会打印一次非错误诊断并回退到 `primary`。
  - `SubHanging.lua` 为 Masterkey 和 M203 增加 `quickAttachmentTransform.muzzleOutletOffset = { x = 226, y = -20 }`。
  - HK416 下挂 flash/spark 拆到 `statuseffecttags="Gunsmith_BarrelParticle"` 的 `StatusEffect`，不再使用 `Distance="50"`。
  - HK416 下挂 `reload=0.4` 保留，但不再通过 XML 写固定 `barrelpos="234,-4"`。
- 新增客户端粒子位置 patch。
  - 只处理带 `Gunsmith_BarrelParticle` tag 的 `StatusEffect`。
  - 粒子位置读取当前 `RangedWeapon.BarrelPos`。
  - 使用 `item.body.DrawPosition` 和 `item.body.DrawRotation` 对齐可见枪身绘制坐标，避免和物理 body 坐标混用。
- 统一 QAT canvas 坐标转换。
  - `GunsmithApi.TryCanvasPointToItemLocal` 纳入 `world.offset`，并以 C# composed world sprite state 作为唯一转换权威。
  - quick-slot transform service 不再读取 Lua 传入的 canvas origin。
  - `Runtime.lua` 不再本地计算 barrel item local position，只传 canvas point 给 C#。
  - HK416 删除手动 quick-slot origin 校准，裸枪口继续直接使用 `muzzle_mount.anchor`。
- 保持 V0.1 原则。
  - 没有新增静默 `try/catch`。
  - 非 GunSmith 或无 QAT barrel transform 走原版 fallback。
  - malformed payload、非有限数值、无效 `BarrelPos` 会显式报错。

## 验证重点

- HK416 无 muzzle、短 muzzle、长 muzzle/消音器时，主枪口发射点和粒子位置分别等价旧 `234/264/324,26` 行为。
- HK416 切换到 Masterkey / M203 时，下挂第一发从 visible lower rail weapon muzzle 发射，并且 muzzle flash 贴合。
- HK416 从下挂切回主弹时，第一发立即回到主枪口。
- 快速连射不因开火路径反复写 `BarrelPos` 产生额外压力。
- QuickMod slot 1-5 拖出、拖回、错误物品拒绝后状态正常。
- DeepLaser 起点和手电光源继续贴合可见附件。
- 原版非 GunSmith 枪械不受 QAT barrel patch 影响。

## 验证状态

- 静态检查已通过：GunSmith / Lua / HK416 XML 中没有 `muzzleLength` 或 `projectileIndex` 残留。
- 静态检查已通过：没有 `RangedWeapon.Use` / `SwitchableRangedWeapon.Use` 写入 `BarrelPos` 的 patch。
- V0.4.1 静态检查已通过：HK416 XML 不再写下挂固定 `barrelpos="234,-4"`，下挂 flash/spark 不再使用 `Distance="50"`。
- V0.4.2 静态检查已通过：`Barrel.lua` 不再配置结构枪管 `quickAttachmentTransform`，barrel registry 不再累计父级 outlet offset。
- V0.4.3 静态检查已通过：Lua/C# 功能代码不再保留手动 quick-slot origin 校准，barrel 注册改为 canvas point hook。
- Windows 编译已通过：
  - `GunSmith` client Debug x64 编译通过。
  - `GunSmith` server Debug x64 编译通过。
- 游戏内 HK416 验证已通过：枪口火焰位置、QuickMod、DeepLaser、手电行为正常。
- V0.4.1 游戏内验证已通过：selector bridge 生效，Masterkey / M203 下挂发射点和 tagged flash/spark 会跟随 `lower_rail` rule，不再错误使用主枪口 / 延长枪管位置。
- V0.4.3 仍需游戏内验证：裸枪口、muzzle 配件、DeepLaser、手电和 lower rail 下挂是否全部跟随统一转换。

## 后续工作

- 若要迁移其他武器，需要保证裸枪口 quick-slot anchor 准确，并只为实际安装在 quick slot 上的 muzzle / 下挂武器配置 `quickAttachmentTransform.muzzleOutletOffset`。
- 声音、Explosion、弹壳位置暂不进入 QAT barrel path。
