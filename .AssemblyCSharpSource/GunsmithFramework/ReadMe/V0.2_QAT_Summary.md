# V0.2 QAT Summary

日期：2026-05-20

QAT V0.2 的目标是解除 GunSmith quick-slot 附件对真实 contained item 物理位置的依赖。V0.1 已经让 DeepLaser 和 quick-slot `LightComponent` 读取 transform service；V0.2 继续保留这个 transform 查询层，但不再通过 `GunsmithQuickSlotLayoutPatch` 移动 hidden quick-slot 附件的 body、rect、submarine 或 hull。

## 已完成内容

- `GunsmithQuickSlotLayoutPatch.ApplyLayouts` 不再执行物理布局。
  - 不再调用 `SetTransformIgnoreContacts`。
  - 不再写入 contained item 的 `Rect`、`Submarine`、`CurrentHull`。
  - 不再主动刷新 contained item 的原生 light transform。
- 旧 Lua hook 和配置形状保持不变。
  - `DeepGunsmithRegisterQuickSlotLayout` 继续注册 layout rule。
  - `DeepGunsmithApplyQuickSlotLayouts` 仍可调用，但现在只是兼容入口。
  - `quickSlotBindings` 不需要迁移。
- `SuspendLayouts` / `ResumeLayouts` 保留为 no-op 兼容 wrapper。
  - 现有 QuickMod GUI 调用点不需要同步修改。
  - V0.3 可以再删除这些 wrapper 和调用点。
- transform service 增加非有限数值检查。
  - 如果 GunSmith quick-slot transform 生成 NaN/Infinity 或无效方向，会打印 weapon、attachment、slot 和 transform 数值。
  - 非 GunSmith / 无 transform 的正常 fallback 不受影响。
- quick-slot 可见附件继续由 GunSmith composed world sprite 表达。
  - 本轮不新增 transform-driven quick-slot render patch。
  - 避免和 composed world sprite 中已有的 quick-slot visual layer 重复绘制。

## 验证重点

- HK416 QuickMod slot 1-5：拖出、拖回、错误物品拒绝后状态正常。
- HK416 muzzle：单击安装/拆卸后附件不消失。
- 可见附件仍随 GunSmith 合成枪身正确显示。
- DeepLaser 起点仍贴合可见 laser device。
- 手电光源仍贴合可见手电，左右朝向、移动、瞄准时持续跟随。
- 原版非 GunSmith 枪械的手电和 DeepLaser 行为不变。
- 控制台不应出现 `GunSmith QAT produced an invalid quick attachment transform.`。

## 验证状态

游戏内验证已通过：QAT V0.2 去除 quick-slot 物理位置写入后，HK416 QuickMod、可见附件、DeepLaser 和手电光源功能正常。

## 后续工作

- V0.3 删除 no-op `SuspendLayouts` / `ResumeLayouts` 和 GUI 调用点。
- 若未来要逐件动态绘制 quick-slot 附件，需要先设计 render metadata，并从 composed sprite 中排除对应 quick-slot visual layer。
