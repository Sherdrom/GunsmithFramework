# V0.1 NPC Preset Summary

日期：2026-05-30

本轮目标是让 GunSmith 枪械可以用 NPC XML 直接选择预设改装，而不是在 NPC XML 里手工塞瞄具、握把、枪口件等 quick-slot 配件。最终方案是：NPC 武器 item 写 `gunsmithpreset="..."`，具体改装表放在 Lua 配置里，运行时把 profile 应用到枪械状态，并自动补齐真实 quick-slot 配件 item。

## 已完成内容

- 新增 XML 入口。
  - NPC 武器 item 支持 `gunsmithpreset` 属性。
  - 示例：`<Item identifier="deep_hk416" gunsmithpreset="ghost_hk416_soldier">`。
  - 一个 NPC 同时携带多把 GunSmith 枪时，每把枪各自写自己的 `gunsmithpreset`。
- 新增 NPC preset 配置。
  - 配置入口是 `Gunsmith.Config.npcPresets.profiles`。
  - 当前配置文件位于 `Lua/Scripts/Gunsmith/Config/NPC/NpcPresets.lua`。
  - `Config.lua` 负责加载该配置文件。
  - `Lua/Scripts/Gunsmith/NpcPresets.lua` 只保留运行时逻辑，不再内联具体 NPC profile。
- profile 格式简化。
  - 不再使用单独的 `rules` 表。
  - `profile.weapon` 限定可应用的武器 identifier，避免 XML 写错枪时误应用。
  - `profile.parts` 直接使用 GunSmith selection path 到 part id 的映射。
- 新增 C# bridge。
  - `GunsmithNpcPresetPatch` 从 NPC item XML 捕获 `gunsmithpreset`。
  - 实测 NPC 生成路径全部走 `Item(Rectangle, ItemPrefab, Submarine, bool, ushort)` 构造器。
  - 已删除多余的 world-position constructor patch。
  - 正常路径调试日志已清理，只保留失败/异常日志。
- 新增 quick-slot 实体配件补齐。
  - `NpcPresets.lua` 根据 quick path 找到对应 weapon slot。
  - 使用 part 的 `item.identifier` 生成真实配件 item，例如 `deep_2x5x_sight` 对应 `2.5x_sight`。
  - `GunsmithQuickPartItemSpawner` 把生成的 item 放进枪械 hidden quick slot。
  - 客户端单人和服务端都注册了 `DeepGunsmithEnsureQuickPartItem`。
  - 客户端插入 hidden quick slot 时复用 `BeginQuickSlotMutation/EndQuickSlotMutation`，避免被 hidden slot 保护逻辑拦截。

## 当前 Profile 示例

```lua
local profiles = config.npcPresets.profiles

profiles.ghost_hk416_soldier = {
    weapon = "deep_hk416",
    parts = {
        ["receiver"] = "hk416_receiver_std",
        ["receiver/barrel"] = "hk416_barrel_std",
        ["receiver/handguard"] = "hk416_handguard_std",
        ["receiver/optic_mount"] = "deep_2x5x_sight",
        ["receiver/handguard/Lower_rail_mount"] = "deep_vertical_grip",
    }
}
```

## 关键流程

1. `HumanPrefab.InitializeItem` prefix 读取 NPC item XML。
2. 如果 XML 上有 `gunsmithpreset` 和 `identifier`，把二者加入 pending queue。
3. `Item(Rectangle, ItemPrefab, Submarine, bool, ushort)` postfix 捕获刚生成的 item。
4. C# bridge 按 item identifier 消费 pending preset，并通知 Lua。
5. Lua 查找 `Gunsmith.Config.npcPresets.profiles[presetName]`。
6. Lua 构造 selection，写入 `Gunsmith.State.selections`，并在服务端保存到 `GunsmithData.SavedState`。
7. 对 quick-slot path，Lua 调 C# 生成真实配件 item 并插入武器隐藏槽。
8. `Runtime.Apply` 刷新 GunSmith 视觉、属性、quick-slot 状态。

该流程只扫描短生命周期 pending queue，不遍历全地图 item、NPC inventory 或所有角色。

## 注意事项

- quick-slot 配件需要对应 part 配置里有 `item.identifier`，否则只能影响视觉/属性，不能生成真实 contained item。
- 如果枪上已经有非空 `GunsmithData.SavedState`，服务端不会用 NPC preset 覆盖它。
- `profile.weapon` 应始终填写，避免同名 preset 被错误用到其它武器。
- 后续新增 NPC 枪械时，优先在 `Config/NPC/NpcPresets.lua` 增加 profile，再在 XML 武器 item 上引用 preset。
- 如果 NPC preset 数量继续增长，可以把 `Config/NPC/NpcPresets.lua` 拆成目录入口 + 多个 NPC/武器分文件。
