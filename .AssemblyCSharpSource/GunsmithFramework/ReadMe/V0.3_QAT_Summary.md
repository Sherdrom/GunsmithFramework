# V0.3 QAT Summary

日期：2026-05-20

QAT V0.3 的目标是清理 V0.2 后剩下的 no-op 兼容层，不改变 quick-slot 存储、Lua 配置、transform 查询、DeepLaser 或 `LightComponent` 行为。

## 已完成内容

- 移除 `GunsmithQuickSlotLayoutPatch` 中的空兼容方法。
  - 删除 `ApplyLayouts`、`SuspendLayouts`、`ResumeLayouts`。
  - 删除 `Item.SetContainedItemPositions` postfix。
  - 保留 layout rule 注册和查询，供 transform service 使用。
- 移除 QuickMod GUI 中的 suspend/resume 调用点。
  - 打开 quick overlay 时不再调用 `SuspendLayouts`。
  - 关闭 quick overlay 时不再调用 `ResumeLayouts`。
  - `RestoreBuffersToWeapon()` 保持不变。
- 移除已废弃的 Lua hook 兼容入口。
  - `DeepGunsmithClearQuickSlotLayouts` 和 `DeepGunsmithRegisterQuickSlotLayout` 保持原行为。
  - 删除 C# 侧 `DeepGunsmithApplyQuickSlotLayouts` hook 注册。
  - 删除 Lua 侧 `Hook.Call("DeepGunsmithApplyQuickSlotLayouts", item)` 调用。

## 验证重点

- HK416 QuickMod slot 1-5：拖出、拖回、错误物品拒绝后状态正常。
- HK416 muzzle：单击安装/拆卸后附件不消失。
- 可见附件仍由 GunSmith composed world sprite 正常显示。
- DeepLaser 起点和手电光源继续贴合可见附件。
- 控制台不应出现 `GunSmith QAT produced an invalid quick attachment transform.`。

## 验证状态

- 静态检查已通过：GunSmith C# 中没有 `SuspendLayouts`、`ResumeLayouts`、`ApplyLayouts`、`SetContainedItemPositions` 残留。
- `DeepGunsmithApplyQuickSlotLayouts` 已从当前 C# hook 和 Lua 调用路径移除。
- QAT / DeepLaser 相关源码没有新增 `try/catch`。
- Windows client 编译已通过：
  - `GunSmith` 编译通过；仍有 15 个既有 `ILuaCsHook.Add` obsolete warnings。
  - `DeepLaser` 编译通过；无 warning。
- 游戏内 HK416 QAT 回归验证已通过：QuickMod、可见附件、DeepLaser 和手电光源功能正常。
