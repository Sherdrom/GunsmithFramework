# V0.5 QAT Summary

日期：2026-05-22

QAT V0.5 的目标是把 GunSmith canvas 坐标转换统一收敛到 C# composed world sprite state，移除 HK416 的手动 `itemPosOrigin` 校准。Lua 只提供 GunSmith canvas 上的 quick-slot anchor；C# 负责把该点按世界贴图的 `scale / rotation / offset / origin` 转成 Barotrauma item local 坐标。

## 已完成内容

- 统一 canvas -> item local 转换。
  - `GunsmithApi.TryCanvasPointToItemLocal()` 纳入 `WorldSettings.Offset`，并按 Barotrauma item local 坐标翻转 Y。
  - `GunsmithQuickAttachmentTransformService` 不再读取 Lua 传入的 `CanvasOrigin`。
  - `GunsmithQuickSlotLayoutPatch.QuickSlotLayoutRule` 删除 `CanvasOrigin` 字段。
  - `DeepGunsmithRegisterQuickSlotLayout` 暂时保留旧参数形状，但忽略 origin，避免 Lua/C# 调用点一次性大迁移。
- 迁移 barrel 注册到 canvas point。
  - 新 hook：`DeepGunsmithRegisterQuickAttachmentBarrelCanvasPoint`。
  - Lua 侧不再本地计算 item local `BarrelPos`，只传 quick-slot anchor、`muzzleOutletOffset` 和 world settings。
  - Client 侧使用 composed sprite state 转换。
  - Server 侧使用 Lua 传入的 canvas/world settings 做同一公式转换，保持 `BarrelPos` 写入服务端权威。
- 删除手动原点校准。
  - 删除 `Core.QuickSlotCanvasOrigin()`。
  - 删除 HK416 `roots.receiver.itemPosOrigin`。
  - 删除 `DeepGunsmithSetItemPosOrigin`、`DeepGunsmithNudgeItemPosOrigin`、`DeepGunsmithPrintItemPosOrigin` 调试命令。
  - Validation 和文档不再要求 `itemPosOrigin`。
- 保持配置语义。
  - `hk416_barrel_std.muzzle_mount.anchor = { x = 220, y = 1 }` 保持不变，作为裸枪口出口点。
  - 结构枪管不再配置 `quickAttachmentTransform`。
  - `Muzzle.lua` 和 `SubHanging.lua` 继续用 `quickAttachmentTransform.muzzleOutletOffset` 表达安装部件自身出口偏移。
- 修复 quick attachment transform scale 漏乘。
  - DeepLaser / 手电读取的 quick attachment transform 在 item local -> world/draw position 时会乘 `owner.Scale`。
  - 该修复不改变 `BarrelPos` 路径；枪口粒子仍读取当前 `RangedWeapon.BarrelPos`，保证粒子和弹丸发射点一致。

## 验证状态

- 静态检查已通过：
  - GunSmith Lua/C# 中无功能性 `itemPosOrigin`、旧 debug commands、旧 `DeepGunsmithRegisterQuickAttachmentBarrelTransform` 残留。
  - `Runtime.lua` 不再保留本地 `canvasPointToItemLocal()`。
  - `Barrel.lua` 不再配置结构枪管 `quickAttachmentTransform`。
  - `Gunsmith_BarrelParticle` tag 未改。
  - 新代码没有新增静默 `try/catch`。
- 编译已通过：
  - `GunSmith` Client Debug x64 编译通过；仅有既有 `ILuaCsHook.Add` obsolete warnings。
  - `GunSmith` Server Debug x64 编译通过。
  - `DeepLaser` Client Debug x64 编译通过。
- 游戏内阶段性结论：
  - 裸枪口 / barrel particle 位置已对齐。
  - 测试发现 DeepLaser / 手电同步偏右上；原因是 quick attachment transform 漏乘 `item.Scale`。
  - 已修复 scale 漏乘，仍需进游戏复测 DeepLaser、手电在左右朝向、移动、瞄准时是否贴合。
