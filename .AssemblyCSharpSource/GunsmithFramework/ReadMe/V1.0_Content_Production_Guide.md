# GunSmith V1.0 内容生产指引

V1.0 的目标是让新枪械、新配件的坐标生产更快、更稳定。核心工具是外部单文件编辑器：

```text
.Tools/GunsmithAnchorEditor/index.html
```

直接用浏览器打开即可，不需要安装 Node、Vite、Electron 或 C# 桌面环境。工具不会读取或改写 Lua 文件，只负责导入精灵图、取坐标、预览拼接，并导出可粘贴的 Lua 片段。

## 坐标规则

当前 GunSmith 使用 receiver-led 武器模型：

```text
weapon
  -> receiver
       -> barrel
       -> handguard
            -> top_rail
                 -> optic_mount
       -> pistol_grip
       -> stock
```

坐标含义如下：

- `visual.source`：原始精灵图上的裁切区域，使用图片像素坐标。
- `visual.attachPoint`：当前 source 内的本地连接点，使用 source 局部坐标。
- `weapon.rootSockets.receiver`：平台 canvas 上的机匣根连接点。
- `mount.anchor`：父配件上的子挂点，使用“相对父配件 attachPoint”的坐标。
- `visual.order`：绘制顺序，越小越早绘制。
- `visual.scale`：单个配件图层缩放。

运行时近似规则：

```text
root part draw offset = rootSocket - attachPoint * partScale
child anchor world = parentDrawOffset + (parentAttachPoint + mount.anchor) * parentScale
child draw offset = childAnchorWorld - childAttachPoint * childScale
```

## 推荐素材准备

建议把同一平台或同一武器族的配件放在同一张 PNG 里，方便取点和复用。

推荐准备：

- 完整机匣 source。
- 枪管 source。
- 护木 source。
- 手枪握把 source。
- 枪托 source。
- 导轨、瞄具、前握把等附件 source。

素材最好保持同一朝向、同一像素比例。source 周围可以保留少量透明边，但 attachPoint 必须落在方便对齐的结构连接点上。

## 从零制作一把 AR 枪

1. 打开 `.Tools/GunsmithAnchorEditor/index.html`。
2. 导入精灵图。
3. 点击“填入 M4 示例”，观察现有 M4 的配置如何拼接。
4. 新建或选择 receiver。
5. 使用“框 source”框出机匣。
6. 使用“取 attachPoint”点击机匣连接基准点。
7. 使用“取 rootSocket”在预览画布上放置 receiver 根点。
8. 新增 barrel / handguard / pistol_grip / stock 子部件，并把父部件设为 receiver。
9. 分别框选子部件 source，设置子部件 attachPoint。
10. 使用“取 mount.anchor”在预览画布上点击父机匣上的对应挂点。
11. 观察下方当前武器预览，直到结构件对齐。
12. 点击“生成 Lua 片段”，把 `rootSockets`、`visual`、`mounts` 片段粘到对应配置文件。
13. 进游戏运行 `DeepGunsmithValidate`，再打开 GunSmith UI 做最终确认。

## attachPoint 怎么选

attachPoint 应该选在“这个配件要贴到父配件上的物理接口点”。

常见选择：

- 机匣 receiver：选择平台上最稳定的整枪定位点，通常是枪管/护木前接口附近或你希望 rootSocket 对齐的位置。
- 枪管 barrel：选择枪管尾部与机匣连接的位置。
- 护木 handguard：选择护木后端与机匣连接的位置。
- 手枪握把 pistol_grip：选择握把顶部插入机匣的位置。
- 枪托 stock：选择枪托前端与机匣/缓冲管连接的位置。
- 瞄具 optic：选择瞄具底部与导轨接触的位置。

## mount.anchor 怎么选

mount.anchor 是父配件提供给子配件的挂点。它不是图片绝对坐标，而是相对父配件 attachPoint 的坐标。

例如 receiver 提供枪管挂点：

```lua
{ path = "barrel", accepts = { "AR_barrel" }, anchor = { x = 62, y = -3 } }
```

含义是：从 receiver 的 attachPoint 出发，向右 62 像素、向上 3 像素的位置，是 barrel 的连接点。

## 工具导出后放到哪里

武器配置：

```lua
config.weapons.deep_example = {
    platform = "AR",
    rootParts = {
        receiver = "example_receiver_std"
    },
    rootSockets = {
        receiver = { x = 209, y = 160 }
    }
}
```

配件视觉：

```lua
visual = {
    texture = texture,
    source = { x = 254, y = 370, w = 135, h = 96 },
    attachPoint = { x = 71, y = 47 },
    order = 30,
    scale = 1.0
}
```

父配件挂点：

```lua
mounts = {
    { path = "barrel", name = "枪管", accepts = { "AR_barrel" }, defaultPart = "AR_barrel_std", anchor = { x = 62, y = -3 } },
    { path = "handguard", name = "护木", accepts = { "AR_handguard" }, defaultPart = "AR_handguard_std", anchor = { x = 62, y = -3 } }
}
```

## 最终进游戏检查

外部工具用于快速取点和初步拼接，最终仍以游戏内效果为准。

建议检查：

- `DeepGunsmithValidate` 为 0 errors。
- M4/HK416 默认外观没有被破坏。
- 新武器默认装配显示完整。
- 替换护木、枪管、握把、枪托后，子挂点仍跟随父配件。
- preview、inventory、world 三套显示参数分别调，不互相混用。

## V1.0 不处理的内容

- 不实现 AK 平台。
- 不实现 gameplay stats 生效。
- 不实现多人同步或服务端权威。
- 不实现工作台入口。
- 不自动修改 Lua 文件。
